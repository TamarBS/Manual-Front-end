Volver a JavaScript

Escuchadores

Se denominan Event Handlers o Event Listeners y sirven para manejar y verificar inputs, acciones del usuario o del navegador.

onchange

Un elemento HTML ha sido cambiado.

onclick, onmousedown y onmouseup

El usuario a clickado un elemento HTML y se han llevado a cabo tres eventos diferentes. Onmousedown es le momento de hacer click sobre el elemento. Onmouseup es el momento de levantar el click sobre el elemento. Onclick es la acción de los dos anteriores en el mismo elemento. Por eso a veces cuando se hace click en un botón, pero se arrastra sin soltar y se suelta fuera del área del botón, la acción del botón no se ejecuta. Esto es porque utiliza un evento onclick en lugar de un onmousedown.

onmouseover

El usuario ha colocado el ratón por encima de un elemento HTML.

onmouseout

El usuario a sacado el ratón de encima de un elemento HTML.

onkeydown

El usuario a pulsado alguna tecla del teclado.

onload y onunload

El usuario entra o sale de la página. Es útil para controlar las coockies por ejemplo.

oninput

Se ejecuta en el momento en el que ocurre algo en un input.

onchange

Se ejecuta en el momento en el que se termina de realizar un cambio en un input. Por ejemplo en un input de texto se añade una palabra. Al salir del input se ajecuta el onchange.

Método addEventListener(event, function, useCapture)

Los anteriores son eventos que se pueden ejecutar en una página web realizando alguna acción. Los escuchadores son métodos que lo que hacen es poner la atención, "escuchar", si alguno de estos eventos ocurre en un elemento HTML concreto.

Permiten llevar a cabo varias acciones a la vez de un mismo evento o varias acciones de varios eventos diferentes.

Los parámetros que se pueden pasar a esta función son:

  • evento Es el evento que se espera escuchar. Pueden ser de muchos tipos, desde acciones del reatón y del teclado hasta cambios de forma en la ventana de la página web (resize).

  • function Es la acción que se espera llevar a cabo contenida en una función. No se permiten añadir parámetros a estas funciones, pero sí pueden contener otras funciones que sí lleven sus propios parámetros.

  • useCapture para elementos con escuchadores que están dentro de otros elementos con otros escuchadores existe una manera de controlar qué escuchadores se ejecutan primero.

    • bubbling primero se ejecuta el escuchador del elemento hijo y después el del padre. Va de dentro hacia afuera.

    • capturing primero se ejecuta el escuchador del padre y luego el del hijo. Va de fuera hacia adentro.

Para dejar de "escuchar" se utiliza el método removeEventListener()
