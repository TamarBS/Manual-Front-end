Encontrar elementos HTML

Hay métodos que permiten encontrar y afectar elementos HTML en función de diferentes selectores e identificadores..

getElementById()

Permite seleccionar un elemento HTML por su id.

document.getElementByID( "id" );
getElementsByTagName()

Busca todos los elementos por el nombre de su etiqueta. Ej.; getElementByTagName("p") buscará todas las etiquetas <p>.

document.getElementByTagName( "nombre de etiqueta" );
// Para buscar sólo elementos dentro de otra elemento se puede utilizar este truco. let padre = document.getElementByTagName( "nombre de etiqueta Padre" );
let hijos = padre.getElementByTagName( "nombre de etiqueta hijos" );

HTMLCollection Object

Este método devuelve una colección de objetos HTML, que es algo parecido a un array con todos los elementos que encuentra. Pero no es un array porque no puede utilizarse algunos métodos como valueOf(), push(), pop(),etc. Gracias a esto es posible acceder a uno de los elementos como en un array myCollection[3] y calcular el .length para saber cuántos elementos se han encontrado permitiendo a su vez hacer loops.

Se diferencia de NodeList porque se puede acceder a sus elementos utilizando el nombre de la etiqueta, el id o su posición en la lista, mientras nodeList sólo permite aceder por su posición en la lista.

Además permite acceder a elementos que se crean después, NodeList no

getElementByClassName()

Encuentra todos los elementos con el mismo atributo clase.

document.getElementByClassName( "nombre de clase" )
querySelectorAll()

Encuentra los elementos utilizando selectores de elementos como en CSS.

document.querySelectorAll( "p.intro" )

NodeList object

Este método devuelve una colección de nodos que contienen el atributo de clase con el valor pasado como parámetro. Es muy parecido a un HTMLCollection, pero tiene alguna diferencia. Puede hacerse lo mismo, acceder a sus nodos myCollection[2] y calcular el número de nodos encontrados con .length y hacer loops.

Se diferencia de HTMLCollection porque se puede acceder a sus elementos utilizando sólo su posición en la lista, mientras HTMLCollectiont permite acceder por nombre, id y también su posición en la lista.

Además no permite acceder a elementos que se crean después, HTMLCollection sí puede.
