Los programas JavaScript se ejecutan por el navegador. Se ejecutan de uno en uno en el orden en el que aparecen en el código.

";" al final de cada estado

Un programa de ordenador es una lista de instrucciones para ser ejecutadas por un ordenador. En programación a estas instrucciones se les llama estados (statements).

Cáda línea de código es un estado. Todos los estados tienen que terminar con punto y coma. En algunos casos no es obligatorio pero siempre es recomendable ponerlo.

Particularidades

Validación de formularias

Aunque el HTML trae por defecto una serie de ventanas emergentes para corregir algunos problemas de validación de los inputs, con Javascript se pueden controlar todas las posibilidades de validación de los formularos.

La validación es el proceso de asegurar que los inputs que rellena el usuario son correctos y útiles.

  • Server side validation lo lleva a cabo el servidor una vez que el formulario se ha enviado.

  • Client side validation lo valida el navegador antes de que el formulario se haya validado.

En los elementos HTML se pueden añadir una serie de atributos que ayudan a hacer lo que se denomina constarint validation, validación de restricciones.

  • disabled desabilita el input

  • max define un valor máximo para el input

  • min define un valor mínimo para el input

  • required indica que es obligatorio rellenar el input

  • type especifica el tipo de input y le aporta estilos predefinidos adaptados al tipo de contenido que da.

  • pattern define el patrón que seguirá el input. Por ejemplo, un patrón de contraseña muestra los contenidos del input como asteriscos.

En CSS se pueden controlar los estilos de los diferentes estados de los input utilizando los siguientes Pseudo selectores.

  • :disabled Aplica estilos a los elementos con atributo disabled

  • :invalid Aplica estilos a los elementos con valores del input incorrectos, no validos.

  • :optional Aplica estilos a los elementos de un formulario que no tienen atributo required.

  • :required Aplica estilos a los elementos que tengan el atributo required.

  • :valid Aplica estilos a los elementos con valor de sus inputs correcto y válidos.

DOM (Document Object Model)

El Modelo de objeto de documento es el estándar para acceder a los documentos y sus contenidos.

Cuándo una página se carga en el navegador, este genera un documento interno a modo de árbol de nodos con todo lo que contiene. Esto hace posible acceder al contenido de los documentos y modificarlo con JavaScript (y otros lenguajes de programación).

Se divide en tres partes diferentes:

  • Core DOM modelo estándar para todo tipo de documentos.

  • XML DOM modelo estándar para los documentos XML.

  • HTML DOM modelo estándar para los documentos HTML.

El objeto document representa tu página web. A continuación algunos ejemplos de cómo se utiliza el objeto documento para manipular elementos HTML.

document.getElementByID( "id" )
document.getElementByTagName( "nombre de etiqueta" )
document.getElementByClassName( "nombre de clase" )
HTML DOM

Es el modelo estándar de objeto e interfaz de programación (son los métodos y propiedades de cada objeto) que permite aplicar cambios en los elementos HTML. Define:

  • Los elementos HTML como objetos.

  • Las propiedades de todos los elementos HTML.

    Son los valores de los elementos HTML que puedes cambiar. Ej.: .innerHTML es una propiedad que permite cambiar el contenido de un elemento HTML.

  • Los métodos para acceder a todos los elementos HTML.

    Son las acciones que se pueden llevar a cabo en los elementos HTML. Ej.: getElementByID es un método que permite acceder a un elementoHTML a través de su ID.

  • Los eventos para todos los elementos HTML.

Nodos

Los nodos son los contenidos del documento y hay que resaltar como importante no confundirlos con elementos HTML. Nodo ≠ elemento HTML.
Por ejemplo un elemento HTML p con contenido es un nodo, pero su texto y sus atributos pertenecen a otro nodo tipo texto o nodo tipo atributo. De manera que si quiero acceder al texto del elemento no me valdrá con acceder al nodo del párrafo, sino que necesito acceder al nodo hijo del párrafo que es el nodo de texto dónde sí que encontraré el texto contenido.

Relación entre nodos

El primer nodo que contiene todos se denomina root. Nunca tiene un padre.Es el principio de todo el documento.

Padre es el nodo que contiene otros nodos. Excepto el root todos los nodos tienen un padre. El padre del primer nodo padre es root.

Un nodo hijo es aquel que está después del padre. El padre contiene al hijo.

Existen dos maneras de llamar a los dos hijos más destacados que son el primero de la lísta y el último de la lista. firtChild y lastChild.

Sibling (hermanos) son nodos que comparten el mismo padre.

Existen dos maneras de moverse a través de los hermanos con nextSibling que es el siguiente hermano y previousSibling que es el hermano anterior.

Navegar entre nodos

Para acceder a los nodos del DOM existen varias propiedades:

  • parentNode busca al nodo padre a través de un nodo hijo.

  • childNodes[nodenumber] busca el hijo de un elemento que es el padre en este caso en función de su posición, como los valores de un array. El nodenumber = 0 es el primer hijo.

  • firstChild encuentra el primer hijo de la lista desde un elemento que es el padre en este caso.

  • lastChild encuentra el último hijo de la lista desde un elemento que es el padre en este caso.

  • previousSibling se mueve al hermano que está por encima en la lista desde su hermano posterior.

  • nextSibling se mueve al hermano que está por detrás en la lista desde su hermano anterior.

IMPORTANTE RESALTAR: que el contenido de los nodos no son el nodo en sí. El texto contenido en un elemento no es el nodo del elemento sino su hijo tipo nodo de texto.

Para acceder al texto de un nodo de texto se pueden utilizar las propiedades .innerHTML o .nodeValue

Nombre de variables

Se denominan identificadores o nombres a los nombre de variables, funciones y palabras clave.

Para crear un nombre de identificador hay que cumplir las siguientes condiciones:

  • No se permite usar - como separador de palabras.

  • Tienen que comenzar siempre con letras de la a-z o A-Z, el símbolo $ o el símbolo _. Se distinguen mayúsculas de minúsculas. No es lo mismo y que Y. Los símbolos $ y _ los identifica como letras.

  • No puede empezar con números ni ningún otro símbolo.

  • Todas las variables tienen que tener su nombre identificador único.

  • No se pueden usar como identificadores las palabras reservadas.

El símbolo del dolar $ no se utiliza normalmente, pero sí para los alias de la función principal en una librería JavaScript.

El _ no se suele utilizar pero sí para alias privados y variables ocultas.

La posición del array empieza en 0

La primera posición de un array es 0, la segunda 1, la tercera 2...

Límite de caracteres por línea

Se aconseja no hacer líneas de más de 80 caracteres y pasar a la siguiente línea. Cuándo una línea se pasa de caracteres se recomienda cortarla por alguno de sus operadores.

Distingue mayúsculas de minúsculas

Para nombrar variables la convención es empezar con una palabra en minúsuclas y seguir la siguiente con la primera en mayúsculas. Ejemplo.: firstName, lastName.

return

Es una palabra reservada que se convertirá en el útlimo paso de la función. No hará nada el programa después del return. Sirve para alojar y mandar los resultados de la función.

Mostrar objetos por pantalla

Mostrar un objeto por pantalla con un innerHTML dará de resultado: [object Object]

Esto no es lo que se espera cuándo se quiere mostrar un objeto, pero hay diferentes maneras de conseguirlo:

  • Por el nombre de las propiedades: Accediendo a cada una de las propiedades del objeto que se quieren mostrar creando una cadena de texto con sentido.

    const person = {
    name: "John",
    age: 30,
    city: "New York",
    };
    person. name + "," + person. age + "," + person. city;
    // Resultado: John, 30, New York

  • Bucle de propiedades: Se puede acceder a las propiedades de un objeto creando un bucle que lo recorra y pinte cada uno de sus valores.

    const person = {
    name: "John",
    age: 30,
    city: "New York",
    };
    let text = "";
    for ( let x in person ) {
    text += person[x] + " ";
    };
    document.getElementById( "demo").innerHTML = text;
    // Resultado: John 30 New York

  • Object.values() : Crea un array de las propiedades del objeto.

    const person = {
    name: "John",
    age: 30,
    city: "New York",
    };
    const myArray = Objetc.values(person);
    document.getElementById( "demo" ).innerHTML = myArray;
    //Resultado: John,30,New York

  • Object.entries() : Método para hacer bucles y recoger tanto las propiedades como los valores.

    const person = {
    name: "John",
    age: 30,
    city: "New York",
    };
    let text = "";
    for ( let [people, value ] of Object.entries(person) ) {
    text += people + " " + value + ",";
    };
    document.getElementById( "demo").innerHTML = text;
    // Resultado: name John, age 30, city New York,

  • JSON.stringify() : Método de JSON que convierte objetos en cadenas de texto (strings). Este método funciona en JavaScript y lo soportan los navegadores.

    const person = {
    name: "John",
    age: 30,
    city: "New York",
    };
    document.getElementByID( "demo" ).innerHTML = JSON.stringify(person);
    // Resultado: {"name":"John","age":30,"city":"New York"}

this

valores

Hay dos tipos de valores:

fijos / literales

  • números: no llevan comillas.

    Se pueden escribir con anotación científica. Ej.: 123e5 = 12300000, 123e-5 = 0,00123

    El máximo número de decimales es 17. No hace correctamente algunos cálculos con decimales, así que se aconseja multiplicar para convertir en números enteros. Ej.: 0,2 + 0,1 = 0,3000000000004, Resolver multiplicando por 10. (0,2*10 + 0,1*10 ) /10

    JavaScript intentará hacer el cálculo de número incluso cuándo sean string con todos los operadores aritméticos excepto con +.

    NaN

    Significa not a number. No es un número. Es una palabra reservada.

    Puede utilizar la función isNaN() para descubrir si un valor es NaN.

    Infinity -Infinity

    Devolverá este valor los cálculos que se pasen del límite de números máximos.

    Hexadecimales

    Existe un código para números no numérico que comienza siempre con 0x seguido de letras. Ej.: 0XFF = 255

    Cambiar base

    Por defecto la base de los números es base 10, pero se puede cambiar la base con el método toString() con el parámetro de la base a aplicar. Ej.: cambiar el número 7 a una base binaria: toString(2) = 111;

  • strings: Siempre llevan comillas. Son cadenas de texto. Los número que van entre comillas son strings y son tratados como palabras.

    Los strings siempre van entre comillas, se pueden usar 'comillas simples' o "comillas dobles" indistintametne. Si se quiere que se muestre en pantalla alguna comilla, no podrá coincidir con la comilla que se esté usando para agrupar el string. Ej.: "quiero poner 'comillas'", 'quiero poner "comillas"'

    Para el caso en el que se necesite utilizar los dos tipos de comilla se puede utilizar la comilla ` llamada template. Es el acento francés. Ej. `Necesito usar 'comillas simples' y "comillas dobles" en este string`

    También hay otras soluciones para esto mismo sustituyendo las comas por \', \", \\, Ejemplo.: Quiero poner \'comillas\'. Quiero poner \\una barras lateral.

    Dentro de las comillas simples se pueden incluir interpolaciones de variables y expresiones dentro de los strings.

    let firstName = "John";
    let texto = `Bienvenido ${firstName}`; // Interpolacion de variable
    let texto = `Bienvenido ${firstName + 1*5.toString()}`; // Interpolacion de expresión.
    // La sintaxis de una interpolación en un template es ${...}

  • undefined : El valor no trae nada, está vacío.

variables

Se utilizan para contener datos. Crear variables se denomina "declarar" y es buena práctica hacerlo al principio del script.

automático
nombreVariable= "valor";

No se declara, pero esto no es lo recomendable. Se recomienda declarar siempre las variables.

var
var nombreVariable = "valor";

Está anticuado, ya se puede utilizar sólo let and const. Reservar su úso solo para código destinado a ser ejecutado en navegadores antiguos.

Particularidades

  • Tiene alcance global. Se puede acceder a la variable aunque esté dentro de llaves {}.

    var nombreVariable;
    function nombreFunción () {
    nombreVariable = "valor"; // Funciona porque es global.
    }

  • Permite re-declarar.

  • Se pueden declarar después de asignar un valor.

    nombreVariable = "valor";
    var nombreVariable ;
let
let nombreVariable = "valor";

Es la forma moderna de var. Sus valores pueden cambiar.

Particularidades

  • Tiene alcance de bloque (block scope), no global o de función. Block scope quiere decir que las declaraciones que van dentro de llaves {} no se pueden acceder desde fuera.

    let nombreVariable;
    function nombreFunción () {
    nombreVariable = "valor"; // NO funciona porque NO es global.
    }

  • No se puede re-declarar en el mismo scope, pero sí en diferentes scopes.

    let nombreVariable = "valor";
    function nombreFunción () {
    let nombreVariable = "valor"; // Funciona porque está en diferente scope
    }
    let nombreVariable = "valor"; // No funciona porque está en el mismo scope que la primera declaración.

  • Se tiene que declarar antes de ser usada.

    <!-- Esto NO funcionará -->
    nombreVariable = "valor";
    let nombreVariable ;

    <!-- Esto Sí funcionará -->
    let nombreVariable ;
    nombreVariable = "valor";

  • No se pueden vincular con this.

  • Se pueden declarar varias variables en un mismo let separando con comas.

    let nombreVariable = "valor", nombreVariable = "valor", nombreVariable = "valor";
const
const nombreVariable = "valor";

Valores constantes que no van a cambiar ni va a cambiar su tipo de valor. No define un valor constante, sino que define una referencia constante a un valor. Por eso no se pueden reasignar valores, arrays u objetos, pero sí cambiar elementos de arrays u objetos.

Es la variable que se utiliza para declarar arrays y objetos.

const cars = ["Saab", "Volvo", "BMW"];
cars[0] = "Toyota"; // Es correcto

cars = ["Toyota", "Volvo", "BMW"]; // Es incorrecto

Es decir puede cambiar elementos específicos pero no reasignar valores al identificador.

Particularidades

  • Tiene alcance de bloque (block scope), no global o de función. Block scope quiere decir que las declaraciones que van dentro de llaves {} no se puede acceder desde fuera.

    const nombreVariable;
    function nombreFunción () {
    nombreVariable = "valor"; // NO funciona porque NO es global.
    }

  • No se puede re-declarar en el mismo scope, pero sí en diferentes scopes.

    const nombreVariable = "valor";
    function nombreFunción () {
    const nombreVariable = "valor"; // Funciona porque está en diferente scope
    }
    const nombreVariable = "valor"; // No funciona porque está en el mismo scope que la primera declaración.

  • Se tiene que declarar antes de ser usada.

    <!-- Esto NO funcionará -->
    nombreVariable = "valor";
    const nombreVariable ;

    <!-- Esto Sí funcionará -->
    const nombreVariable = "valor";

  • No se pueden vincular con this.

  • Se le debe asignar un valor en el momento de ser declarada, no después.

    <!-- Esto NO funcionará -->
    const nombreVariable ;
    nombreVariable = "valor";

Destructuring

Lo normal es declarar la variable y depués del = indicar los valores, pero en el caso del destructuring se declaran primero los valores y después del = el nombre de la variable.

Esto sirve para llegar a los valores de un objeto de forma más rápida.

Primero se declara un objeto y después se utiliza el destructuring para acceder a alguna de las propiedades del objeto.

// Crea el objeto. En este caso un Array
const fruits = ["Bananas", "Oranges", "Apples", "Mangos"];
// Destructuring
let [fruit1,,,fruit2] = fruits;
// Añadiendo comas "," está saltando posiciones del array, 2 posiciones en este caso.
// Resultado: Bananas Mangos

    Este operador ternario hace una comparación entre la edad a ver si es mayor o menor de edad. Si la comparación es true devolverá el primer string colocado después de ? si es false devuelve el segundo strong después de :

Al comparar número y strings el resultado será false porque el string no numérico tiene valor de Nan lo que lo hace siempre false. Aunque si el string es numérico intenta hacer el cálculo a modo de orden numérico cifra a cifra. Ej.: "25" será mayor que "100" porque 2 > 1.

    Un set es un espacio de memoria que puede contener valores que no se repiten. Que no se puedan repetir es la característica especial de este objeto. Es de tipo object. No pueden existir dos valores iguales.

    Los set no tienen "indice" sólo valor.

    Es posible crear un set con este método new Set([...]); conteniendo un array, o también creándolo y dándole luego los valores. variable.add("valor");

    Métodos

    Se puede utilizar un for/of para acceder a los valores del set.

    • has() : devuelve true si el valor del argumento está entre los valores del set. Sintaxis: setName.has("d");

    • forEach() : como un array recorre sus valores.

    • values() : devuelve un objeto que se puede iterar con los valores del set.

    • keys() : también devuelve un objeto que se puede iterar con los valores del set para que sea compatible con otros métodos, por ejemplo Maps().

    • entries() : al no tener set valores de índice, este método devuelve un objeto que se puede iterar con valores en pareja [valor,valor]. En los arrays devuelve [índice, valor]. Este método lo hace compatible con métodos como Maps().

Son secuencias de caracteres que forman patrones de búsqueda. Se pueden utilizar para:

  • Buscar textos

  • Reemplazar textos

  • Validar textos

Normalmente se utilizan con métodos de string.

Sintaxis

/pattern/modifier flags;

/palabra/i

  • / /g global. Hace la búsqueda en todo el string.

    / /i case-insensitive. No hace diferencia entre mayúsculas o minúsculas.

    / /m selección múltiple. Devuelve todas las coincidencias que encuentre en el string.

    / /s permite que la palabra esté unida a un punto "." para ser encontrada.

    / /v unicode suport. Incluye en la búsqueda todos los caracteres unicode.

    / /y búsqueda sticky.

Caractér de clase

Va entre [] y coincide con cualquiera de los caracteres que se contienen entre los corchetes.

this

Es una palabra clave de tipo objeto. Se utiliza para que el método al que se quiere apuntar se refiera a sí mismo.

Dependiendo de dónde se encuentre el this, se referirá a diferentes cosas:

  • En el método de un objeto: Se refiere al objeto.

    Sólo: Se refiere al objeto global.

    En una función: Se refiere al objeto global (el objeto que llama a la función).

    En modo estricto: Su valor es undefined.

    En un evento: se refiere al elemento que recibe el evento.

    En un arrow function: No funciona igual que en una función normal. En arrow function siempre se referirá al objeto global (El objeto "dueño" de la función. Siempre será el mismo).
