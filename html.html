HTML

Lenguaje de marcado de hipertexto. Es el lenguaje base en la creación de páginas web. Fue inventado por Tim Berners-Lee en 1991.

Sintaxis

El lenguaje está formado por los elementos HTML.

< etiqueta atributo = "valor" > contenido... </ etiqueta >
Elemento vacío

Existe otra sintaxis para elementos que no ofrecen contenido, por ejemplo un <br>.

< etiqueta >
<!-- Se permite utilizar el símbolo "/" como etiqueta de cierre en los elementos vacíos --> < etiqueta />

Particularidades

No discrimina mayúsculas y minúsculas

Se pueden utilizar mayúsculas y minúsculas, aunque el consenso es usar minúsculas.

Los atributos no necesitan ir entrecomillados

Pero el consenso es que sí aparezcan entre comillas. Da problemas no ir entrecomillado si hay espacios entre palabras. Las comillas por defecto son ("") dobles, pero en caso de que el valor quiera mostrar esas comillas, se pueden usar entonces las simples ('') y viceversa.

Formas de vincular estilos CSS

Se puede vincular al CSS de tres maneras:

Inline

Dentro del elemento HTML como atributo

< etiqueta style = "propiedad:valor; propiedad:valor;" > </ etiqueta >

Interno

Inyectado en el head

< head >
< style >
.className {
propiedad : valor-css ;
}
</ style >
</ head >

Externo

Enlazado a un documento externo .css

< head >
< link rel = "stylesheet" href = "styles/styles.css" >
</ head >

La prioridad es la siguiente, 1 el que más prioridad tiene:

  1. Inline
  2. Internal (A menos que los estilos estén colocados encima del link dentro del head. En este caso External tiene más prioridad.)
  3. External
  4. Los por defecto de cada navegador

XHTML

Es una versión más estricta de HTML. (X de eXtensible). Basicamente, lo que en HTML se puede obviar pero se hace porque son buenas prácticas, en XHTML es obligatorio. Ej.: Comenzar con <!DOCTYPE>, cerrar etiquetas, elementos siempre en minúsculas, las anidaciones tienen que ir en orden estricto, etc.

Enrutado

Se utilizan las siglas URL para referirnos a la dirección de una página web y también la ruta a un archivo. Significa Localizador uniforme de recursos.

Sintaxis

protocolo :// prefijo . dominio : puerto / ruta / (nombre del archivo)
  • protocolo: define el tipo de servicio de Internet (http, https, ftp o file)
  • prefijo: por defecto para http es www
  • dominio: nombre del dominio
  • puerto: número del puerto, para http normalmente es 80
  • ruta: la ruta entre carpetas que lleva al archivo
  • nombre del archivo: archivo en cuestión al que se quiere llegar.

Una URL puede tener un dominio en palabras, o mediante un IP (Internet Protocol) Ej.: 192.68.20.50

Sólo usa el código ASCII, si contiene otros caracteres diferentes se tienen que sustituir. Ej.: no admite espacios, se sustituyen por "+" o "%20". Otros caracteres se sustituyen por "% seguido de el digito que le corresponda al símbolo". Ej.: ® = %C2%AE

Hay dos formas de hacer una ruta a los archivos

Ruta absoluta

Enlaza a un dominio externo a la página web. Al no depender de nuestro mantenimiento puede desaparecer.

Ej.: 'https://www.w3schools.com/html/imagen.jpg'

Ruta a nuestra propia web

Cuando nuestra web está publicada en internet podemos acceder a los archivos del proyecto utilizando una url que empieza con el símbolo "/" que representa el directorio raiz. Se recorren todas las carpetas desde la que contiene todo el proyecto hasta la que contiene el recurso indicado.

Ej.: Mi dominio es https://www.misitio.es

<img src="/carpeta/archivo.jpg">

La imagen está apuntando a la ruta: https://www.misitio.es/meloinvento/lasimagenes/carpeta/archivo.jpg

Ruta relativa

El archivo está alojado dentro de nuestro proyecto.

La ruta se piensa desde el archivo en el que se está hasta al que se quiere llegar.

Entrar en carpetas es nombrarlas seguidas del símbolo "/"

Salir de carpetas es "../"

Ejemplo práctico

  • archivo1
  • carpeta1
    • archivo2
    • archivo3
    • carpeta2
      • archivo4
      • archivo5
  • carpeta3
    • archivo6
    • archivo7

  • Estoy en el archivo3 y quiero ir al archivo5

    Estoy dentro de la carpeta1, así que sólo tengo que entrar en la carpeta2, dónde ya puedo acceder al archivo5

    'carpeta2/archivo5'

  • Estoy en el archivo4 y quiero ir al archivo1

    Salgo de dos carpetas, la carpeta2 en la que estoy y luego la carpeta1 y ya tengo acceso al archivo1

    '../../archivo1'

  • Estoy en el archivo5 y quiero ir al archivo7

    Salgo de dos carpetas, la carpeta2 en la que estoy y luego la carpeta1. Entro en carpeta3 y ya tengo acceso al archivo7.

    '../../carpeta3/archivo7'

  • Estoy en el archivo1 y quiero ir al archivo4

    Entro en la carpeta1, y luego en carpeta2 dónde ya accedo al achivo4

    'carpeta2/archivo5'
