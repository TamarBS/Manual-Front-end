Formulario

Recoge la información que mete el usuario en el formulario para enviarla a una base de datos.

Sintaxis de un formulario

< form >
< label for = "ftext" > input text </ label >
< input type = "text" id = "ftext" name = "ftext" >
</ form >

Lista de Elementos HTML de un formulario

label

Letrero. El lector de pantalla lo lee cuándo el usuario hace focus en el input al que acompaña. Sirve de area clicable del input al que acompaña. El valor del atributo "for" del label y el "id" del input tiene que ser el mismo para poder relacionarlos.

input

Campo para recoger la información aportada por el usuario.

Tipos de input

Dependiendo del valor de su atributo "type" su aspecto y funcionalidad varía.

text

Campo de texto corto.

radio

Permite seleccionar sólo uno de varias opciones añadiendo un name común a todas las opciones. Su nombre viene de los botones de las radios antiguas en las que pulsando uno de los botones se deseleccionaba el anterior.

checkbox

Permite selección múltiple.

button

Opción para añadir un botón que haga alguna acción.

submit

Botón de enviar formulario.

range

Selector interactivo de un rango de cifras.

number

Campo de texto que sólo acepta cifras. Se pueden añadir max y min.

color

Selector de color.

date

Campo de texto sólo para fechas. Algunos navegadores incluyen un date-picker para poder seleccionar más facilmente las fechas.

Atributos min y max con valores en formato fecha para acotar las fechas que se pueden seleccionar. Si la fecha elegida no está dentro de las fechas posibles aparece un aviso al incluir los datos y también al pulsar submit.

datetime-local

Campo de texto para fecha y hora.

email

Campo de texto especial para direcciones de correo web que valida si los valores incluidos parecen una dirección de correo o no

file

Botón para seleccionar archivos. Al pulsar abre la carpeta de archivos local.

hidden

Para incluir en el envío del formulario datos que son necesarios pero el usuario no necesita ver. Nunca usar este elemento para temas de seguridad porque aunque en la web no se ve, se puede acceder desde el inspector.

image

Permite utilizar una imagen como botón de submit

month

Campo de texto especial sólo para seleccionar mes y año.

password

Campo que oculta el contenido

reset

Botón que vacía los campos del formulario

search

Un campo buscador. Es basicamente lo mismo que un campo de texto pero aparece una x que borra el contenido si se pulsa

tel

Campo de texto especial para números de teléfono. Con el atributo pattern se indica el formato aceptado.

time

Campo de texto especial para hora, minuto y segundo.

url

Campo de texto especial para url que validará si lo introducido parece o no una url real.

week

Campo de texto especial para seleccionar un número de semana y un año.

Atributos del input

checked

Indica que una opción será preseleccionada. Por ejemplo en campos como radio o checkbox.

disabled

No se puede editar. Se verá en gris. Es un campo inactivo.

max

Valor máximo del input. Pueden ser cifras, fechas, horas, etc.

maxlength

Máximo número de caracteres permitidos en un campo de texto.

min

Valor mínimo del input. Pueden ser cifras, fechas, horas, etc.

pattern

Indica el patrón que se acepta en el input. Por ejemplo en teléfono se puede añadir para aceptar sólo un formato de espacios, límite de números y símbolos, etc.

readonly

Campo que no se puede editar, sólo se puede leer.

required

Campo obligatorio. El formulario no se enviará si estos campos obligatorios no tienen información.

size

Indica el ancho del campo medido en ancho de cada caracter.

step

En campos numéricos indica el salto de unidades que hace al pulsar en las flechitas de más y menos. Funciona para los input type = number, range, date, datetime-local, month, time y week.

value

El valor por defecto para ese input.

placeholder

Texto que sirve de ejemplo para el contenido del campo. Se puede usar para dar pistas al usuario del formato que puede utilizar en el campo. Ese valor no se envía en el formulario y desaparece cuándo el usuario rellena.

autofocus

Especifica que el campo aparecerá como seleccionado al cargar la página. Sólo debería haber un autofocus en el primero de los campos del formulario, así es muy fácil empezar a rellenar sin tener que andar clickando.

autocomplete

Para dar la oportunidad a autocompletar un elemento sin que sirva para todo el form. Algunos navegadores tienen bloqueada esta función.

form

Este atributo sirve para relacionar un input que está fuera de la etiqueta form y hacer que igualmente se envíe en los datos del formulario, aunque esté fuera. Es necesario que el atributo "form" del input tenga el mismo valor que el id del form.

formaction

Igual que el atributo "action" del elemento form, sirve para especificar a qué documento.php se enviará. Sobreescribe el action del elemento form y sólo funciona en los input type = submit y image.

formenctype

Igual que el atributo "enctype" del elemento form, define el encriptado de la info del formulario. Sólo sirve con el método "post" y con los input type = submit y image. Tiene tres posibles valores:

  • application/x-www-form-urlencoded: Todos los caracteres se codifican antes de ser enviados, es el por defecto.

  • multipart/form-data: Es necesario si el usuario envía un archivo a través del formulario.

  • text/plain: Los datos se mandan sin ninguna codificación. No se recomienda por temas de seguridad.

formmethod

Igual que el atributo "method" del elemento form, sirve para especificar si el formulario se envía en method = post o get. Funciona sólo en input type = submit y image.

formtarget

Igual que el atributo "target" del elemento form, define dónde se abre la respuesta tras pulsar enviar el formulario. Funciona sólo en input type = submit y image.

formnovalidate

Igual que el novalidate, para que no tenga que validar el contenido de un campo.

search

La etiqueta engloba el form y el input search sólo para indicar al navegador que se trata de un buscador. Es una etiqueta semántica.

select

Permite seleccionar varias opciones que se definen cada una dentro de un elemento < option>.

Existe la etiqueta "optgroup" que sirve para agrupar opciones. El valor se le da en un atributo label.

< select name = "cars" id = "cars" >
< optgroup label = "Coches alemanes" >
< option value = "volvo" > Volvo </ option >
</ optgroup >
</ form >

Atributos

  • selected: La primera opción será la seleccionada por defecto. Para elegir otra opción por defecto se le añade "selected".

  • size: En la etiqueta select con valores numéricos. Añade de alto el número de opciones elegido. Si se elige un size de 3 el alto del select dejará ver, sin hacer scroll ni despegar, tres opciones.

  • multiple: Permite seleccionar más de una opción pulsando ctrl

textarea

Campo de texto con espacio de alto para varias frases.

Atributos

  • rows: Con valor numérico define el número de líneas de texto que ocupará el campo de texto.

  • cols: Define el ancho del campo de texto.

fieldset

Enmarca el formulario y con la etiqueta "legend" añade un título. El fieldset va dentro de la etiqueta form, y dentro del fieldset van los demás elementos del formulario.

Legend




datalist

Es algo así como un select (dropdown) pero que está relacionado con un input y por lo tanto permite también añadir texto. Hace como de buscador entre las opciones que contiene.

< form >
< input list = "browsers" >
< datalist id = "browsers" >
< option value = "..." >
< option value = "..." >
</ datalist >
</ form >

El input y el datalist comparten valor en sus atributos list y id.

output

Muestra el resultado de una operación (como las llevadas a cabo por un script).

+ = 100
< form oninput = "x.value=parseInt(a.value)+parseInt(b.value)" >
<-- El script hace que se sume a + b y se guarde en una variable x -->
< input type = "number" id = "a" name = "a" value = "50" >
+
< input type = "number" id = "b" name = "b" value = "50" >
=
< output name = "x" for = "a b" > 100 </ output >
</ form >

Atributos

name

Los campos del formulario tienen que tener un atributo "name" único (excepto radio en el que sólo se selecciona una opción, por eso comparten name) para poder ser enviados.

accept-charset

Especifica el tipo de caracteres que se acepta

rel

Especifica una relación entre un recurso y el documento actual (nidea de lo que significa esto)

enctype

Sólo para metodo "post". Indica cómo se debe encriptar los valores recogidos al mandarlos.

Atributos en la etiqueta <form>

action

Define dónde se enviarán los datos recogidos por el Formulario al pulsar submit.

< form action = "/action_page.php" >
< label for = "ftext" > input text </ label >
< input type = "text" id = "ftext" name = "ftext" >
< input type = "submit" value = "Submit" >
</ form >

Si no se especifica el action, los datos son enviados a la própia página.

target

Define dónde se abre la respuesta tras pulsar enviar el formulario.

method

Define el método para enviar la info. Puede ser "get" (que envía los datos a traves del texto de la url) o "post"(lo envía dentro del cuerpo de la solicitud HTTP)

Diferencias entre get y post

  • Get no es seguro porque los datos son visibles en la url. Post sí es seguro porque se envía oculto.
  • Get tiene límite de 2048 caracteres en la url. Post no tiene límite
  • Get se puede guardar como favorito, post no lo permite.
autocomplete

Define si permite autocompletado o no. Sus valores son: "on" "off". Algunos navegadores tienen bloqueada esta función.

novalidate

Si aparece significa que los datos recogidos no hace falta validarlos. No tiene valores, es sólo poner la palabra.
