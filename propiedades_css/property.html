Regla @property

Se utiliza para reasignar nuevos valores a propiedades CSS sin necesidad de JavaScript.

Sintaxis

Consta de tres propiedades: syntax (tipo de propiedad que se va a afectar), inherits (indica si sus hijos heredarán o no el valor afectado) y initial-value (el valor que se quiere reasignar).

@property --myColor {
syntax: "<color>";
inherits: true;
initial-value: lightgray;
}

Para utilizar la variable @property creada se llama dentro de un var().

Funcionalidad

Con esta variable CSS se puede conseguir cambiar el valor de las propiedades como con :root y var() pero incluso extendiendo sus posibilidades.

Al tener que especificar el tipo de dato se previenen errores.

El initial-value define un valor por defecto para esa propiedad, por lo que si en algún otro selector el valor tiene algún error, aún así se seguira aplicando la propiedad por defecto, teniendo una especia de propiedad plan B.

El valor de si es heredable o no se puede controlar.

Lo interesate de esa variable es que hace cosas que no se pueden hacer con var():

  • reasignar valor fuera de la clase en la que está la propuedad definida

    @property --my-bg-color {
    syntax: "<color>";
    inherits: true;
    initial-value: lightgrey;
    }
    div {
    background-color: var(--my-bg-color);
    }
    .fresh {
    --my-bg-color: red;
    }

  • Crear animaciones

    Junto con @keyframe se puede conseguir animar el css.

    Ejemplo

    @property --startColor {
    syntax: "<color>";
    inherits: false;
    initial-value: purple;
    }
    @property --endColor {
    syntax: "<color>";
    inherits: false;
    initial-value: white;
    }
    .ex1 {
    background: linear-gradient(var(--startColor),var(--endColor));;
    animation: gradient 3s linear infinite;
    }
    @keyframes gradient {
    0%, 100% {
    --startColor: purple;
    --endColor: white;
    }
    50% {
    --startColor: white;
    --endColor: purple;
    }
    }
