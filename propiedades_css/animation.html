Animaciones con CSS

Propiedades que permiten animar elementos HTML sin necesidad de aplicar Javascript. Permite cambiar tantos estilos de los elementos se quiera las veces que se quiera.

Es necesario definir los keyframes.

@keyframes nombreDeKeyframe: {
0% { background-color: red; }
25% { background-color: yellow; }
50% { background-color: blue; }
100% { background-color: green; }
}
div {
widht: 100px;
height: 100px;
background-color: red;
animation-name: nombreDeKeyframe;
animation-duration: 4s;
}

@keyframes

Define que estilos tendrá el elemento en momentos concretos. Permite definir el momento del tiempo total de la animación en la que se ejecutará el cambio de estilo. En el ejemplo de código anterior se ve como se especifica como un nombre de clase los valores con porcentaje % que asignan el momento de aplicación del estilo.

En lugar de estos valores de porcentaje % se pueden usar las palabras from y to que indican 0% y 100%, inicio y final.

Para que estos cambios de estilo se identifiquen con un elemento en concreto al @keyframes lo acompaña una palabra libre que habrá que relacionar dentro de las propiedades del elemento. En el ejemplo el nombre de los keyframes se vincula con la propiedad animation-name en las propiedades del div.

animation-name

Es la propiedad que se pone en el elemento y vincula ese elemento con uno de los keyframes. Tienen que compartir el mismo nombre.

animation-duration

Define el tiempo total en segundos que durará la animación. 1s

animation-delay

Define el tiempo de espera antes de que la animación se arranque. Los valores se miden en segundos. 2s

El valor puede ser negativo, eso hará que la animación arranque justo en el momento que ya han pasado esos dos segundos, "comiéndose" los cambios de estilo que hubiera antes de esos dos segundos. Es decir si tengo una animación de 4s de duración pero le pongo un delay de -2segundos, la animación arrancará en el punto de la animación del keyframe 50%. Si había otros cambios de estilo antes, no se verán. Es algo así como saltarse una parte de la animación.

animation-iteration-count

Define el número de veces que se tiene que repetir la animación antes de que se pare. Puede ser un valor numérico o la palabra infinite para hacer un loop infinito.

animation-direction

Esta propiedad permite reproducir las animaciones en diferente orden. Puede tener los siguentes valores:

  • normal: es el valor por defecto. La dirección es la programada.

  • reverse: la animación se reproduce inverso a como fue programada, cómo rebobinando.

  • alternate: primero se reproduce normal y se rebobina.

  • alternate-reverse: primero se reproduce rebobinando y luego normal.

animation-timig-function

Especifica la curva de velociad del effecto de la animación.

Valores

  • ease: es el por defecto. Empieza y termina despacio y en el medio se acelera.

  • linear: la misma velocidad de inicio a fin.

  • ease-in: sólo tiene inicio lento.

  • ease-out: sólo tiene final lento.

  • ease-in-out: Es como ease pero tarda más en arrancar y parar y no se acelera tanto entre medias.

  • cubic-bezier(n,n,n,n): permite crear una curva de transición totalmente libre.

animation-fill-mode

Las animaciones CSS no aplican estilos al elemento hasta que no se arranca el primer milisegundo de la animación y al terminar deja de afectarle. Para poder afectar antes y después de animar existe esta propiedad. Puede contener los siguientes valores:

  • none: es el valor por defecto. No aplicará ningún estilo antes ni después de que arranque la animación.

  • forwards: el elemento estará afectado por los efectos del final de la animación.

  • backwards: el elemento estará afectado por los efectos del principio de la animación.

  • both: el elemento estará afectado por los efectos del inicio y del final de la animación.

animation-play-state

Especifica cuándo una animación está reproduciéndose(running) o está en pausa(paused).

animation

Es la propiedad aglutinadora.

animation: nombreDeKeyframe duración tipoDeAceleración retraso repetición dirección
