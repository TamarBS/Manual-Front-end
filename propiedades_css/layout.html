Disposición de elementos

Algunas propiedades sirven para definir el comportamiento de los elementos con respecto a otros. Se aplica un orden específico, una colocación dentro de la página y ocupan un espacio determinado.

display

Esta propiedad se utiliza para definir cómo se verá el elemento en la pantalla.

Cada elemento tiene su propio valor por defecto, que puede ser block o inline.

Valores

  • inline: ocupan el ancho del contenido solamente. No se le puede añadir ancho, alto ni márgenes al contenedor. ( span a img )

  • block: ocupan todo el ancho de pantalla. ( div headings p form header footer section )

  • inline-block: Igual que inline, pero permite aplicar márgenes, alto y ancho al contenedor.

  • contents

    Hace desaparecer el contenedor del elemento al que se aplica, ascendiendo a sus hijos a un nivel superior en el DOM. El contenedor desaparece pero no los contenidos.

    Padre
    Hijo
    Nieto
  • flex

    Convierte el elemento en un contenedor que ocupa todo el ancho disponible y permite contener elementos que se sitúen de manera flexible dentro de él.

    Horizontal

    Vertical

  • inline-flex: Cómo flex, pero sin ocupar todo el ancho disponible. Sólo el ancho del elemento. Se convierte en un contenedor y permite contener elementos que se sitúen de manera flexible dentro de él.

  • grid

    Es como el flex pero no actúa solo para una fila de elementos, sino para una cuadrícula de filas y columnas.

  • inline-grid: Es como grid, pero no ocupa todo el ancho disponible, sólo el ancho de su contenido.

  • none: El elemento no se ve. Desaparece y deja de ocupar su sitio. Se suele utilizar junto con JavaScript para ocultar elementos sin tener que eliminarlos. Si se quiere ocultar un elemento pero que respete su sitio mejor utilizar visibility:hidden;

  • initial: Le aplica su valor por defecto.

  • inherit: Hereda su valor del padre.

  • list-item: Aplica el mismo comportamiento a cualquier elemento que si fuera un li dentro de una lista.

  • run-in: No encontré mucha info, pero por lo visto se adapta según el display que tienen los elementos de alrededor.

  • Valores para tablas Estos valores de display, sólo son útiles para tablas.

    • inline-table: Permite colocar una tabla dentro de un texto sin que la tabla ocupe todo el ancho, sólo el ancho de su contenido. Los elementos table por defecto son display="block".

    • table: El elemento actúa como una tabla. Supongo que display block.

    • table-caption: El elemento actúa como el título de una tabla.

    • table-column-group: El elemento se comporta como un colgroup.

    • table-header-group: El elemento se comporta como un thead de tabla.

    • table-footer-group: El elemento se comporta como un tfoot de tabla.

    • table-row-group: El elemento se comporta como un tbody de tabla.

    • table-cell: El elemento se comporta como un td de tabla.

    • table-column: El elemento se comporta como un col de tabla.

    • table-row: El elemento se comporta como un tr de tabla.

position

Define el método de posicionamiento para un elemento.

  • static: Es el valor por defecto. Se posiciona acorde al flujo normal de la página. No le aplican las propiedades top, right, bottom, left.

    static

  • relative: Relativo a su posición normal. Le afecta las propiedades top, right, bottom, left.

    relative left:16px

  • fixed: El elemento se queda fijo en la pantalla y no ocupa espacio. Le afectan las propiedades top, right, bottom, left y no responde al scroll. Como si se quedara pegado en una parte de la pantalla.

    fixed top-right:20px

  • absolute: Se posiciona relativo al elemento ancestro más cercano que esté posicionado. Si no tiene ningún padre, utiliza el body como referencia a modo de último recurso. No ocupa espacio, como fixed. El ancestro posicionado también puede ser otro elemento absolute padre. Un elemento posicionado es todo aquel que tenga propiedad position mientras su valor no sea static.

    padre posicionado.

    absolute top-left 20px

  • sticky: Tiene un comportamiento que mezcla relative y fixed. Da la impresión de que al hacer scroll se queda pegado a una parte de la pantalla. Se necesita mínimo uno de los valores top, right, bottom, left que indicará en qué momento del scroll se va a quedar pegado.

    sticky bottom 0

z-index

Esta propiedad define el orden de pila del elemento. Permite posicionar elementos por encima de otros. Puede contener valores positivos o negativos.

La propiedad sólo funciona si va acompañada de un elemento con position absolute, relative, fixed o sticky. También funciona en los hijos de un contenedor display flex (flex items).

overflow

Controla lo que le pasa al contenido de un elemento que es demasiado grande y no cabe en su sitio, se desborda. Permite cortar o añadir barras de scroll.

Esta propiedad sólo funciona en elementos block con un alto definido.

  • overflow-x: afecta sólo al eje x.

  • overflow-y: afecta sólo al eje y.

  • overflow-anchor

    Permite anclar el texto en la posición de scroll. Esto es útil para que el texto no se baje si se genera un contenido nuevo. Por ejemplo, en un texto, un botón crea rectángulos arriba del todo.

    overflow-anchor: none;

  • overflow-wrap

    Define si las palabras largas que se desbordan de su contenedor se pueden o no cortar y pasar a la siguiente línea.

    Valores

    • normal: las palabras largas se desbordan

    • anywhere / break-word: se supone que son diferentes y break-word tiene en cuenta espacios y guiones para separar por palabras, pero a mí no me lo hace, para mí funcionan igual. Hacen que la palabra larga se corte ajustándose al contenedor y dividiéndose en la siguiente línea.

Valores

  • visible: El valor por defecto. El contenido no se corta y se desborda de su contenedor.

  • hidden: El contenido se recorta y se deja de ver lo que esté desbordado.

  • scroll: El contenido se corta y lo que esté desbordado se puede ver utilizando las barras de scroll que esta propiedad añade. Añade tanto barra de scroll vertical como horizontal, aunque no se necesite.

  • auto: lo mismo que scroll, pero sólo añade las barras de scroll si hacen falta.

float

Propiedad que permite colocar elementos a la derecha o izquierda de otros.

Valores

  • left: el elemento se coloca a la izquierda del todo de su contenedor.

  • right: el elemento se coloca a la derecha del todo de su contenedor.

  • none: Es el por defecto. No se coloca de ninguna manera diferente a la esperada.

  • inherit: hereda la propiedad float de su padre.

Flex box

Flexible Box Layout module. Módulo de diseño de cajas flexibles. Sirve para organizar objetos en filas o columnas sin necesidad de utilizar las propiedades float o position.

Existe otro método de diseño llamado Grid que también cumple con esta función de organización de objetos. La diferencia es que flex funciona de manera unidireccional, es decir, horizontal o vertical y Grid funciona para las dos a la vez, es decir, permite crear cuadrículas de elementos con filas y columnas.

Consta siempre de: flex container y flex items.

Propiedades flex-container

display:flex

Propiedad y valor del contenedor para poder utilizar las propiedades flex.

flex-direction

Dirección de los flex-items. row | columna | row-reverse | column-reverse

flex-wrap

Determina si los contenidos se ajustarán al ancho (nowrap) o pasarán a la siguiente fila (wrap). Existe un valor que permite alterar el orden normal de los flex-items ponendo los objetos que se desbordan sobre los primeros objetos de la fila ( wrap-reverse ).

flex-flow

Es una propiedad aglutinadora para flex-direction y flex-wrap.

flex-flow: flex-direction flex-wrap;
justify-content

Define la manera de organizar los flex-items unos con respecto a otros, con respecto al espacio entre ellos y entre los márgenes derecho e izquierdo.

Los valores pueden ser:

  • center: Los flex-items se colocan centrados dejando márgenes a los lados.

  • flex-start: Es el por defecto. Se alinean de izquierda a derecha, dejando el margen por la derecha.

  • flex-end: Se alinean de derecha a izquierda, dejando el margen por la izquierda.

  • space-around: El espacio se reparte entre los flex-items.En los márgenes puede haber menos espacio que entre los flex-items.

  • space-between: No hay margenes laterales y el espacio se reparte todo entre los flex-items.

  • space-evenly: El espacio total se reparte de manera equitativa tanto en márgenes laterales como entre los flex-items.

align-items

Se utiliza cuándo el alto de los flex-items no ocupa todo el alto de su contenedor, para controlar su posición vertical.

Valores

  • center: Se colocan en el centro del alto de su contenedor sin ocupar todo el alto posible.

  • flex-start: Se colocan arriba del todo.

  • flex-end: Se colocan abajo del todo.

  • stretch: Cambia el valor del alto de los flex-items y pasan a tener de alto lo mismo que su contenedor.

  • baseline: Se alinea con respecto al texto. Es decir, no importa que los flex-items tengan diferentes alturas, cogerán como referencia la base del texto y todos los flex-items compartirán la misma posición para su base de texto.

    1

    2

    3

    4

  • normal: Igual que stretch. Es el por defecto. Se adapta a todo el alto de su contenedor.

align-content

Define la manera de colocar los flex-items cuándo hay múltiples líneas con respecto al alto del contenedor. Existen los siguientes valores:

  • center: Los flex-items se colocan centrados dejando márgenes arriba y abajo.

  • stretch: Es el por defecto. Se reparte entre el número de filas el alto total de los flex-items ocupando lo máximo posible respetando una separación entre filas.

  • flex-start: Se alinean arriba, dejando el margen por abajo.

  • flex-end: Se alinean abajo, dejando el margen por arriba.

  • space-around: El espacio se reparte entre los flex-items.En los márgenes puede haber menos espacio que entre los flex-items.

  • space-between: No hay margenes top-bottom y el espacio se reparte todo entre los flex-items.

  • space-evenly: El espacio total se reparte de manera equitativa tanto en márgenes top-bottom como entre los flex-items.

Propiedades flex-item

Los hijos directos de un contenedor display:flex; pasan directamente a ser flex-items.

order

Especifica el orden de aparición de los flex-items para los casos en los que no se quiera respetar el orden del código.

El valor son números y el por defecto es 0 que permite que se coloquen en orden del código.

flex-grow

Define cuánto va a crecer un flex-item con respecto a los demás.

El valor es un número y el 0 es el por defecto que permite que todos compartan el mismo ancho si no se especifica nada diferente.

El valor 0 hace que no sea posible hacerlo crecer.

flex-shrink

Shrink significa encoger. Define cuánto se va a encoger los flex-items para encajar en su contenedor. Esta propiedad permite impedir que unos flex-items se encojan como los demás.

El valor es un número siendo el por defecto 1, que permite que todos los flex-items se encojan lo mismo.

El valor 0 hace que no sea posible encogerlos.

flex-basis

Especifica el ancho inicial de un flex-item pero no con respecto a sus compañeros, sino en px.

flex

Es una propiedad aglutinadora para flex-grow, flex-shrink y flex-basis.

align-self

Permite afectar la alineación vertical de un sólo flex-item con respecto al alto del contenedor. Puede sobrescribir los valores de align-items del contenedor, es decir, tiene más prioridad.

Grid Layout Module. Módulo de diseño en cuadrícula con filas y columnas.

Consta siempre de grid-container y grid-items.

Propiedades grid-container

Elementos con estilo display:grid | inline-grid;. Esta propiedad permite hacer una organización de contenidos en filas y columnas, como una cuadrícula.

grid-template-columns y grid-template-rows, número de filas y columnas

Define la cantidad de filas y columnas que tendrá el grid.

Admite los valores px, %, em y fr(fraction). Fraction calcula el tamaño en función del resto de grid-items, por lo que es una buena opción para hacer grids responsive.

Si en un grid de cuatro columnas se añaden más de cuatro grid-items automáticamente se crea una segunda fila y se añade a continuación.

Propiedades

  • grid-template-rows: Define el número de filas y puede definir también el tamaño concreto de cada fila. Los valores son una lista separada por espacios en el que cada valor indica una fila y su tamaño. El valor auto asigna el mismo tamaño para todas las filas.

    Ejemplo: grid-template-rows: auto auto;

    *En caso de tener asignados valores a dos filas y crearse grid-items para ser tres filas, en la tercera fila no se aplica el último valor de la propiedad, sino el valor por defecto, auto.

  • grid-template-columns: Define el número de columnas y puede definir también el tamaño concreto de cada collumna. Los valores son una lista separada por espacios en el que cada valor indica una columna y su tamaño. El valor auto asigna el mismo tamaño para todas las columnas.

    Ejemplo: grid-template-columns: 100px 20px;

  • grid: Propiedad aglutinadora.

justify-content

Igual que en flex, organiza los grid-items horizontalmente.

Al ancho total sumado de todos los grid-items tiene que ser menor al ancho total para que esta propiedad haga algún efecto.

  • center: Los grid-items se colocan centrados dejando márgenes a los lados.

  • start: Se alinean al inicio del ancho disponible.

  • end: Se alinean al final del ancho disponible.

  • space-around: El espacio se reparte entre los grid-items.En los márgenes puede haber menos espacio que entre los grid-items.

  • space-between: No hay margenes laterales y el espacio se reparte todo entre los grid-items.

  • space-evenly: El espacio total se reparte de manera equitativa tanto en márgenes laterales como entre los grid-items.

Propiedades grid-item

Todos los hijos directos de un grid-container pasan a ser grid-items.

grid-column y grid-row tamaño

Imaginando una cuadrícula con las líneas que separan los cuadrados numeradas de izqueirda a derecha y de arriba a abajo nos hacemos una idea de cómo están asignadas las coordenadas de cualquier grid.

Pensar en esos números nos ayudará a utilizar las siguientes propiedades:

  • grid-cloumn-start: Especifican dónde empieza un grid-item de manera horizontal. Su valor es un número que corresponde a alguna línea vertical de la cuadrícula.

  • grid-column-end: Especifican dónde acaba un grid-item de manera horizontal. Su valor es un número que corresponde a alguna línea vertical de la cuadrícula.

  • grid-row-start: Especifican dónde empieza un grid-item de manera vertical. Su valor es un número que corresponde a alguna línea horizontal de la cuadrícula.

  • grid-row-end: Especifican dónde termina un grid-item de manera vertical. Su valor es un número que corresponde a alguna línea horizontal de la cuadrícula.

  • grid-column: Es una propiedad aglutinadora de grid-column-start y grid-column-end.

    grid-column: start / span end ;

  • grid-row: Es una propiedad aglutinadora de grid-row-start y grid-row-end.

    grid-row: start / span end ;
grid-area

Es una propiedad aglutinadora para grid-row-start, grid-column-start, grid-row-end y grid-column-end.

grid-area: 1/ 1/ span 4/ span 1;

La propiedad permite crear un nombre y luego utilizarlo en otra propiedad grid-template-areas como unidad de medida.

.item1 {
grid-area: myArea;
}
grid-container {
grid-template-areas: ' myArea myArea myArea myArea myArea ';
}

justify-self

Sirve para ajustar el ancho del grid-item a su espacio en el grid. Permitirá añadir espacios vacíos dentro de su celda, colocando el item de diferentes maneras.

  • auto: valor por defecto. heredado del pariente.

  • normal: Como stretch a veces y otras como start.

  • stretch: Ocupa todo el ancho posible.

  • start: Se coloca al principio.

  • left: Se coloca a la izquierda.

  • center: Se coloca centrado con espacio a los lados.

  • end: Se coloca al final.

  • right: Se coloca a la izquierda.

  • overflow-alignment: como start, pero reconoce si el contenido se desborda y elige la mejor opción.

  • baseline alignment: Alineado con la baseline del padre.

align-self

Sirve para ajustar el alto del grid-item a su espacio en el grid. Permitirá añadir espacios vacíos dentro de su celda, colocando el item de diferentes maneras.

  • auto: valor por defecto. heredado del pariente.

  • stretch: Ocupa todo el alto posible.

  • flex-start: Se coloca al principio.

  • center: Se coloca centrado con espacio arriba y abajo.

  • flex-end: Se coloca al final.

  • baseline: Alineado con la baseline del padre.
