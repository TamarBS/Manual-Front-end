Disposición de elementos

Algunas propiedades sirven para definir el comportamiento de los elementos con respecto a otros. Se aplica un orden específico, una colocación dentro de la página y ocupan un espacio determinado.

display

Esta propiedad se utiliza para definir cómo se verá el elemento en la pantalla.

Cada elemento tiene su propio valor por defecto, que puede ser block o inline.

Valores

  • inline: ocupan el ancho del contenido sólamente. No se le puede añadir ancho, alto ni márgenes al contenedor. ( span a img )

  • block: ocupan todo el ancho de pantalla. ( div headings p form header footer section )

  • inline-block: Igual que inline, pero permite aplicar márgenes, alto y ancho al contenedor.

  • contents

    Hace desaparecer el contenedor del elemento al que se aplica, ascendiendo a sus hijos a un nivel superior en el DOM. El contenedor desaparece pero no los contenidos.

    Padre
    Hijo
    Nieto
  • flex

    Convierte el elemento en un contenedor que ocupa todo el ancho disponible y permite contener elementos que se situen de manera flexible dentro de él.

    Horizontal

    Vertical

  • inline-flex: Cómo flex, pero sin ocupar todo el ancho disponible. Sólo el ancho del elemento. Se convierte en un contenedor y permite contener elementos que se situen de manera flexible dentro de él.

  • grid

    Es como el flex pero no actúa solo para una fila de elementos, sino para una cuadrícula de filas y columnas.

  • inline-grid: Es como grid, pero no ocupa todo el ancho disponible, sólo el ancho de su contenido.

  • none: El elemento no se ve. Desaparece y deja de ocupar su sitio. Se suele utilizar junto con JavaScript para ocultar elementos sin tener que eliminarlos. Si se quiere ocultar un elemento pero que respete su sitio mejor utilizar visibility:hidden;

  • initial: Le aplica su valor por defecto.

  • inherit: Hereda su valor del padre.

  • list-item: Aplica el mismo comportamiento a cualquier elemento que si fuera un li dentro de una lista.

  • run-in: No encontré mucha info, pero por lo visto se adapta según el display que tienen los elementos de alrededor.

  • Valores para tablas Estos valores de display, sólo son útiles para tablas.

    • inline-table: Permite colocar una tabla dentro de un texto sin que la tabla ocupe todo el ancho, sólo el ancho de su contenido. Los elementos table por defecto son display="block".

    • table: El elemento actúa como una tabla. Supongo que display block.

    • table-caption: El elemento actúa como el título de una tabla.

    • table-column-group: El elemento se comporta como un colgroup.

    • table-header-group: El elemento se comporta como un thead de tabla.

    • table-footer-group: El elemento se comporta como un tfood de tabla.

    • table-row-group: El elemento se comporta como un tbody de tabla.

    • table-cell: El elemento se comporta como un td de tabla.

    • table-column: El elemento se comporta como un col de tabla.

    • table-row: El elemento se comporta como un tr de tabla.

position

Define el método de posicionamiento para un elemento.

  • static: Es el valor por defecto. Se posiciona acorde al flujo normal de la página. No le aplican las propiedades top, right, bottom, left.

    static

  • relative: Relativo a su posición normal. Le afecta las propiedades top, right, bottom, left.

    relative left:16px

  • fixed: El elemento se queda fijo en la pantalla y no ocupa espacio. Le afectan las propietades top, right, bottom, left y no responde al scroll. Como si se quedara pegado en una parte de la pantalla.

    fixed top-right:20px

  • absolute: Se posiciona relativo al elemento ancestro más cercano que esté posicionado. Si no tiene ningún padre, utiliza el body como referebcua a modo de último recurso. No ocupa espacio, como fixed. El ancestro posicionado también puede ser otro elemento absolute padre. Un elemento posicionado es todo aquel que tenga propiedad position mientras su valor no sea static.

    padre posicionado.

    absolute top-left 20px

  • sticky: Tiene un comportamiento que mezcla relative y fixed. Da la impresión de que al hacer scroll se queda pegado a una parte de la pantalla. Se necesita mínimo uno de los valores top, right, bottom, left que indicará en qué momento del scroll se va a quedar pegado.

    sticky bottom 0

z-index

Esta propiedad define el orden de pila del elemento. Permite posicionar elementos por encima de otros. Puede contener valores positivos o negativos.

La propiedad sólo funciona si va acompañada de un elemento con position absolute, relative, fixed o sticky. También funciona en los hijos de un contenedor display flex (flex items).

overflow

Controla lo que le pasa al contenido de un elemento que es demasiado grande y no cabe en su sitio, se desborda. Permite cortar o añadir barras de scroll.

Esta propiedad sólo funciona en elementos block con un alto definido.

  • overflow-x: afecta sólo al eje x.

  • overflow-y: afecta sólo al eje y.

  • overflow-anchor

    Permite anclar el texto en la posición de scroll. Esto es útil para que el texto no se baje si se genera un contenido nuevo. Por ejemplo, en un texto, un botón crea rectángulos arriba del todo.

    overflow-anchor: none;

  • overflow-wrap

    Define si las palabras largas que se desbordan de su contenedor se pueden o no cortar y pasar a la siguiente línea.

    Valores

    • normal: las palabras largas se desbordan

    • anywhere / break-word: se supone que son diferentes y break-word tiene en cuenta espacios y guiones para separar por palabras, pero a mí no me lo hace, para mí funcionan igual. Hacen que la palabra larga se corte ajustándose al contenedor y dividiéndose en la siguiente línea.

Valores

  • visible: El valor por defecto. El contenido no se corta y se desborda de su contenedor.

  • hidden: El contenido se recorta y se deja de ver lo que esté desbordado.

  • scroll: El contenido se corta y lo que esté desbordado se puede ver utilizando las barras de scroll que esta propiedad añade. Añade tanto barra de scroll vertical como horizontal, aunque no se necesite.

  • auto: lo mismo que scroll, pero sólo añade las barras de scroll si hacen falta.

float

Propiedad que permite colocar elementos a la derecha o izquierda de otros.

Valores

  • left: el elemento se coloca a la izquierda del todo de su contenedor.

  • right: el elemento se coloca a la derecha del todo de su contenedor.

  • none: Es el por defecto. No se coloca de ninguna manera diferente a la esperada.

  • inherit: hereda la propiedad float de su padre.

De alguna manera float hace que elementos block como un div se hagan inline y sea posible colocar varios uno al lado del otro de forma vertical.

Div sin float. El texto del elemento aprovecha cada espacio vacío de su contendor padre no ocupado por los float. El comportamiento del texto es diferente al de su contenedor. El texto parece reconocer el espacio de los float, mientras el contenedor no. Por eso los float aparecen dentro del div aunque en código estén fuera y se desbordan en el eje y.
Float Right dentro

Tiene un comportamiento especial cuándo va seguido de elementos sin float, porque se posiciona como relative top-left=0 o top-right=0 permitiendo que el texto lo rodee aunque para el contenedor sea como si estuviera en una position absolute, no reconoce su espacio ocupado en la coordenada y, pero sí en su coordenada x.

Clearfix

Para que no ocurra el desbordamiento se puede añadir un overflow:auto; al contenedor para que reconozca el alto del elemento float. Esto también hará visual la diferencia de elementos tal cual se define en código. Si están fuera o dentro, separandolos en columnas. A este "hack" se le conoce como Clearfix.

Float left
Float Right
Div sin float con overflow:auto; El texto se desplaza a la derecha dejando espacio al float:left; y también respeta el espacio para el float:right que contiene. El float:right; es más alto que el contenedor y se desborda, porque el contenedor no reconoce su height pero sí su width.
Float Right dentro del elemento sin float

Puede ocurrir que al añadir overflow:auto; al contenedor aparezcan barras de scroll no deseadas. Para conseguir el resultado deseado se pueden utilizar la siguiente clase.

.clearfix ::after {
content: "";
clear: both;
display: table;
}

clear

Permite colocar elementos debajo de otros que tengan alguna propiedad float.

Valores

  • none: Es el por defecto y no afecta en nada al elemento.

  • left: Se coloca debajo de los elementos con float:left;

  • right: Se coloca debajo de los elementos con float:right;

  • both: Se coloca debajo de los elementos con float tanto left como right.

  • inherit: Hereda la propiedad clear de su pariente.
